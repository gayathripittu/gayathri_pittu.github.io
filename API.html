Function-Based Views

This early project employed function-based views to manage passenger information effectively, providing basic CRUD (Create, Read, Update, Delete) functionalities.
The project enabled users to add new passenger entries, inputting essential details such as name, age, gender, and contact information directly into the database. Additionally, it provided easy access to a comprehensive list of passengers, allowing for quick reference and management. Users could update existing passenger records to maintain accuracy over time and remove passengers from the system as necessary, promoting more efficient data management practices.

Class-Based Views

Expanding from my knowledge of function-based views (FBVs), this project explores Django's class-based views (CBVs). CBVs provide a structured and efficient way to manage typical view tasks, making development easier and promoting code reuse.
This Django CBV Course Management System demonstrates the effectiveness of CBVs. The project featured Course Creation for entering new course details like name, instructor, and duration. They could access a comprehensive list of all courses and update existing ones with ease. Additionally, the system enabled the removal of outdated or irrelevant courses.
